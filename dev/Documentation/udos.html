BLUESIM contains a number of entities available for simulations.

Click on the tabs above to learn more information about how each entity functions within a BLUESIM simulation.


Definition


A source is an entity that generates the data that flows through the simulation. This can either be static data or custom User Defined Objects if set by the user. Sources should be linked to other entities to initialize the flow of data in the simulation.


Parameter(s)

  • Distribution

Definition


A queue is an entity that holds a set of data generated by the source that must exit in the same order they come in. Upon running the simulation, each queue will generate a set of downloadable data. The data displayed will be the average number of arrivals, number of drops, server utilization, time spent in queue, as well as the total number of customers in the system.


Parameter(s)

  • Distribution
  • Maximum Queue Length: Maximum amount of data that a queue may hold before it overflows.
  • Number of Servers: The number of servers used in the queue. This is amount of data that can be processed concurrently.


Definition


A split function is an entity that takes in data from a single path and outputs it to two possible paths depending on the specified fraction of traffic that flows to the top channel. For example, an input of 0.5 would send 50% of traffic to the top channel, resulting in an equal distribution of data sent to the top and bottom. If UDOs are being used, the UDO field can be specified and more advanced functions can be used to determine the fraction of traffic sent to each channel.


Parameter(s)

  • Fraction of data sent to top channel, or custom function (if UDOs are used)

Definition


A sink is an entity trough which data exits the simulation. Sinks cannot link to other entities as they represent the end point of data. After running the simulation, each sink will generate a set of downloadable data that represents the average number of arrivals, population, and average stay duration of data in the simulation. Sinks require no special parameters.


Parameter(s)

  • None

Definition


A function block is an entity that applies an operation to a User Defined Object. When an object passes through a function, the specific operation is performed on it and the new data is passed along to the next connected entity in the simulation.


Parameter(s)

  • UDO field being used
  • Function to be applied (ex: a value of '+1' will add '1' to the object before passing it through the connection)

Definition


A reverser is an entity that reverses the flow of data for the purpose of creating feedback loops. With a reverser, data can be redirected to pass through a previous entity again for additional filtering. For example, in a “DMV” simulation it may be determined that after waiting in a line, 70% of users can pass through while 30% need to re-do their paperwork. Those 30% are redirected through a reverser to the back of the queue to wait in line again. Reversers require no special parameters.


Parameter(s)

  • None

Definition


A thermometer is an entity that measures statistics about the data that passes through it. Thermometers do nothing to alter the data, and merely serve as a tool for measuring information about the traffic passing through a specific point. Thermometers display statistics about the average population and time spent in the simulation and require no special parameters.


Parameter(s)

  • None

Definition


A cloner is an entity that clones the data that passes through it, and sends each clone on a different path. Cloners do nothing to alter the data, and merely serve as a tool to send the same traffic on two paths. Cloners require no special parameters.


Parameter(s)

  • None

Definition


A stack is an entity that holds a set of data generated by the source that must exit in the opposite order they come in (first in, last out).


Parameter(s)

  • None
